Optimisations

Note that it is possible to pass options to, or disable, any of the bundled transforms. Simply pass the module name as documented here, with an options object to customise the behaviour, or false to disable completely. For example, to remove all comments, you can do:

{
    discardComments: {
        removeAll: true
    }
}

If you would like to keep all comments, you can pass this options object instead:

{
    discardComments: false
}

All examples aside from core are formatted with semicolons and whitespace, to aid readability.

autoprefixer#

Removes unnecessary prefixes based on the browsers option. Note that by default, it will not add new prefixes to the CSS file.

.box {
    -moz-border-radius: 10px;
    border-radius: 10px;
    display: flex;
}
.box {
    border-radius: 10px;
    display: flex;
}

discardOverridden#

Removes at-rules which have the same identifier as another; for example two instances of @keyframes one. As the browser will only count the last of these declarations, all others can safely be removed.

    to {
        opacity: 0;
    }
}

    to {
        transform: rotate(359deg);
    }
}
.box {
    animation-name: one;
}
@keyframes one {
    0% {
        transform: rotate(0deg);
    }
    to {
        transform: rotate(359deg);
    }
}

    animation-name: one;

normalizeCharset#

Ensures that only a single @charset is present in the CSS file, and moves it to the top of the document. This prevents multiple, invalid declarations occurring through nave CSS concatenation. Note that by default, new @charset rules will not be added to the CSS.

    content: "";
}
